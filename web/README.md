# TaskMarket Web

Next.js 14 (App Router, TypeScript) client for the TaskMarket API. The frontend is a thin
client — all business logic lives in the [`/api`](../api) service, reached through the typed
client in `lib/api.ts` (no raw `fetch` in components).

## Stack

Next.js 14 · Tailwind · shadcn/ui (hand-authored, Tailwind v3) · TanStack Query ·
React Hook Form + Zod · Clerk.

> **Versions**: Clerk is pinned to **v6** (v7 needs Next.js 15+, and we target Next.js 14 per
> `CLAUDE.md`). The shadcn primitives in `components/ui/` are hand-authored for Tailwind v3
> rather than generated by the CLI (which defaults to v4).

## Run locally

```bash
npm install
cp .env.example .env.local      # add your Clerk keys (see below)
npm run dev                     # http://localhost:3000
```

The API must be running too (see [`../api/README.md`](../api/README.md)).

### Environment

| Var | Purpose |
| --- | ------- |
| `NEXT_PUBLIC_API_BASE_URL` | FastAPI base URL (default `http://localhost:8000`) |
| `NEXT_PUBLIC_CLERK_PUBLISHABLE_KEY` | Clerk publishable key |
| `CLERK_SECRET_KEY` | Clerk secret key |

Get Clerk keys from [dashboard.clerk.com](https://dashboard.clerk.com). The Razorpay key used by
the checkout widget is returned by the API's `/v1/payments/checkout`, so no Razorpay env is
needed here.

### Signing in

The interactive features (post, offer, chat, pay, review, notifications) need a real Clerk
session to call the authenticated API. Public pages — **browse**, **task detail**, **profiles** —
render without signing in. Without Clerk keys the app still builds and the public pages work; the
signed-in surfaces show a sign-in prompt.

## Pages

| Route | Rendering | What |
| ----- | --------- | ---- |
| `/` | hybrid | Landing + quick links |
| `/browse` | client | Task feed with filters (category, location, city, budget, search, sort) |
| `/post` | client (auth) | Multi-step post wizard: details → location → budget → photos → review |
| `/tasks/[id]` | server + islands | Task detail, offer/payment panel, reviews |
| `/dashboard` | client (auth) | My Tasks + My Jobs & Offers (with withdraw) |
| `/messages` | client (auth) | Conversation list (unread badges, polling) |
| `/messages/[conversationId]` | client (auth) | Message thread (bubbles, composer, polling) |
| `/profile/[id]` | server | Public profile: rating, completion rate, badges, review history |
| `/settings` | client (auth) | Edit profile + avatar + Poster/Tasker toggles |

A global **top bar** (`components/top-bar.tsx`) provides nav + a **notification bell** (polls
unread count) + the Clerk user menu. Real-time-ish updates use TanStack Query polling (no
WebSockets in Phase 1).

## Layout

```
app/
  layout.tsx        # ClerkProvider > Providers > TopBar + page
  providers.tsx     # TanStack Query client
  error.tsx         # error boundary
  <route>/page.tsx  # see table above
components/
  *.tsx             # feature components (wizard, panels, thread, bell, cards, reviews…)
  ui/               # shadcn-style primitives (button, input, card, select, skeleton, …)
lib/
  api.ts            # typed API client (all endpoints + upload helpers)
  format.ts         # INR formatting
  utils.ts          # cn()
middleware.ts       # clerkMiddleware()
```

## Build

```bash
npm run build       # type-check + lint + production build
```
